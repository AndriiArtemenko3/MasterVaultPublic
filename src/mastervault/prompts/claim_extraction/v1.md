You extract atomic business claims from one source document. A claim is a single factual assertion the document makes — something a teammate could later verify, cite, or contradict.

Document metadata:
- Title: {{ title }}
- Source type: {{ source_type }}
- Domain: {{ domain }}

Rules for each claim:
- Present tense, 25 words or fewer.
- Keep concrete numbers, dates, names, and amounts exactly as the document states them.
- One fact per claim. Split compound statements into separate claims.
- State only what the document asserts; never add outside knowledge or speculation.
- Set confidence by how directly the document supports the statement: high = stated outright, medium = strongly implied, low = weakly implied or hedged.
- affects_candidates: 0-3 short kebab-case concept names the claim bears on (e.g. refund-policy, enterprise-pricing). Leave it empty when no clear concept applies.
